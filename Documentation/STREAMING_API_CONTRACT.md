# Streaming API Contract: /chat/stream

This document defines the streaming interface for the `/chat/stream` endpoint.

## Overview
The `/chat/stream` endpoint accepts a JSON POST request and returns a stream of newline-delimited JSON objects (JSON Lines format). This allows the frontend to render responses incrementally.

## Endpoint: `POST /chat/stream`

### Request Body
```json
{
  "message": "string",
  "session_id": "string",
  "avatar": "string"
}
```

### Response Format (Streaming)
The server streams a series of newline-delimited JSON objects. Do **not** attempt to parse the full response body as a single JSON array, as this will fail.

#### 1. Token chunks
Sent for every piece of content generated by the LLM.
```json
{"type": "token", "content": "string"}
```

#### 2. Replace (Guardrail triggered)
Sent if the output guardrail determines the streamed response should be replaced (e.g., due to compliance issues). The frontend should replace the entire current chat bubble content with this new content.
```json
{"type": "replace", "content": "string"}
```

#### 3. Error
Sent if the streaming process fails unexpectedly.
```json
{"type": "error", "content": "string"}
```

#### 4. Termination (Final message)
Sent once the stream is complete.
```json
{"type": "done", "ferpa_blocked": boolean}
```
